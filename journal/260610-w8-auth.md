# W8 — Authentication and API tokens

**Date:** 2026-06-10. **Unit:** W8 (plan §1.3.8, decision 4).

## What was built

Hand-rolled auth (~200 lines, as decided — no JWT, no fastapi-users):

- `src/library/auth/passwords.py` — Argon2id via `pwdlib[argon2]`
  (passlib is dead on 3.13; pwdlib is FastAPI's own current choice).
- `src/library/auth/service.py` — session + API-token lifecycle. Both
  credentials are opaque 256-bit `secrets.token_urlsafe` values stored
  only as SHA-256 hex in the existing W2 `sessions` / `api_tokens`
  tables (no migration needed). Sliding session expiry
  (`LIBRARY_SESSION_TTL_DAYS`, default 30), with refresh writes
  throttled to once per 5 minutes; same throttle for token
  `last_used_at`.
- `src/library/auth/deps.py` — `current_user` (bearer header
  authoritative, else session cookie, else 401 with a generic body) and
  `csrf_protect` (double-submit: `X-CSRF-Token` header must equal the
  non-httpOnly `library_csrftoken` cookie for cookie-authenticated
  POST/PATCH/PUT/DELETE; bearer and safe methods exempt).
- `src/library/api/auth.py` — login (sets `library_session` httpOnly +
  CSRF cookies, `SameSite=Lax`, `Secure` from `LIBRARY_COOKIE_SECURE`),
  logout (deletes the session row), me, token list/create/revoke. Token
  secret (`library_…` prefix) is returned exactly once at creation.
- Whole `/api` surface gated at include level in `app.py`
  (`dependencies=[Depends(current_user), Depends(csrf_protect)]`);
  login is mounted separately as the only open `/api` route; `/healthz`
  stays unauthenticated. Uploads now record `uploader_id`.
- `src/library/cli.py` + `library` console script (typer):
  `user add/passwd/disable/list`. `disable` flips `is_active` and
  revokes all sessions/tokens. Each command opens and disposes a fresh
  NullPool engine (a cached engine would be bound to a dead event loop
  across `asyncio.run` calls).

## Decisions and gotchas

- **Login is CSRF-exempt**: no session exists yet and the password
  itself proves intent; logout is NOT exempt (cookie-authenticated
  POST).
- **`current_user` runs before `csrf_protect`** in the router
  dependency list so anonymous requests get a clean 401, not a
  confusing 403.
- **Generic 401 on login** — wrong password, unknown user, and disabled
  account return byte-identical bodies (no account enumeration); test
  asserts equality of all three.
- **Tests:** `api_client` is now an authenticated fixture (fresh
  random user + real cookie login, CSRF header pre-set from the
  cookie), `anon_client` added. The W3/W7 suites passed unchanged —
  fixture-level churn only. A route-table sweep test asserts every
  `/api` route except `POST /api/auth/login` 401s anonymously, so
  future routers are covered automatically.
- Tests set `LIBRARY_COOKIE_SECURE=false` because TestClient speaks
  plain `http://testserver` and httpx would never return a `Secure`
  cookie; one test flips it to `true` and asserts the flag appears.
